# 墨焕影视 - 管理员账号

## 概述

墨焕影视管理后台通过 `AdminUser` 表管理管理员账号。管理员登录后获取 JWT Token，凭 Token 访问 `/api/admin/*` 系列接口。

- **数据表**：`admin_users`（Prisma 模型 `AdminUser`）
- **登录接口**：`POST /api/auth/admin-login`
- **登录页地址**：`/login`
- **密码加密**：bcrypt（salt rounds 10）
- **角色**：`super_admin`（超级管理员）、`editor`（编辑）

## 默认超级管理员账号

项目通过 Prisma seed 脚本（`packages/api/prisma/seed.ts`）提供默认超级管理员账号，执行 `prisma db seed` 后自动创建：

| 项目 | 值 |
|------|----|
| 用户名 | `Ethen` |
| 邮箱 | `ethen@mhvideo.com` |
| 密码 | `Ethen112..` |
| 角色 | `super_admin` |

> **⚠️ 安全警告**：此为默认账号，密码已公开在源码中。**上线后必须立即修改密码**，详见下文「修改密码」与「安全建议」。

Seed 脚本为幂等执行：按用户名查询，已存在则跳过，不会重复创建也不会覆盖已有密码。执行命令：

```bash
pnpm --filter @mhvideo/api exec prisma db seed
```

## 登录地址

管理后台登录页路径为 `/login`，完整地址：

- 本地开发：`http://localhost:5173/login`（admin 包 Vite 默认端口）
- 生产环境：`https://你的域名/login`

未登录用户访问任何管理页面会自动重定向到 `/login`；已登录用户访问 `/login` 会自动跳转到 `/dashboard`。

## 登录方式

管理员登录支持 **用户名或邮箱** 两种标识，通过 `/api/auth/admin-login` 接口完成。

**请求**：`POST /api/auth/admin-login`

**请求体**：

```json
{
  "account": "Ethen",
  "password": "Ethen112.."
}
```

`account` 字段可为用户名或邮箱：
- 若值包含 `@`，按邮箱查询
- 否则按用户名查询

**登录示例**：

```bash
# 用户名登录
curl -X POST https://你的域名/api/auth/admin-login \
  -H "Content-Type: application/json" \
  -d '{"account": "Ethen", "password": "Ethen112.."}'

# 邮箱登录
curl -X POST https://你的域名/api/auth/admin-login \
  -H "Content-Type: application/json" \
  -d '{"account": "ethen@mhvideo.com", "password": "Ethen112.."}'
```

**成功响应**：

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
    "admin": {
      "id": "clx...",
      "username": "Ethen",
      "email": "ethen@mhvideo.com",
      "role": "super_admin"
    }
  }
}
```

登录成功后，前端将 `access_token` 存入 `localStorage`（键名 `admin_token`），后续请求通过 `Authorization: Bearer <token>` 头携带。Token 默认有效期 7 天，可通过 `refresh_token` 刷新。

## 修改密码

默认密码 `Ethen112..` 必须在上线前修改。提供以下两种方式：

### 方式一：通过 Prisma Studio

1. 在本地 `packages/api` 目录配置好 `.env`（`DATABASE_URL` 指向生产数据库）
2. 启动 Prisma Studio：

```bash
pnpm --filter @mhvideo/api exec prisma studio
```

3. 浏览器打开 `http://localhost:5555`
4. 选择 `AdminUser` 表
5. 找到 `Ethen` 记录，将 `passwordHash` 字段替换为新密码的 bcrypt 哈希值
6. 点击 **Save** 保存

> 生成 bcrypt 哈希可在 Node REPL 中执行：
> ```bash
> node -e "const b=require('bcryptjs');console.log(b.hashSync('你的新密码',10))"
> ```
> 需在 `packages/api` 目录下执行（已安装 `bcryptjs`）。

### 方式二：编写脚本更新

创建一次性脚本（如 `packages/api/scripts/reset-password.ts`），使用 bcrypt 哈希新密码并更新数据库：

```typescript
import bcrypt from 'bcryptjs';
import { PrismaClient } from '@prisma/client';

const prisma = new PrismaClient();

async function main() {
  const username = process.argv[2]; // 传入用户名
  const newPassword = process.argv[3]; // 传入新密码

  if (!username || !newPassword) {
    console.error('用法: tsx scripts/reset-password.ts <用户名> <新密码>');
    process.exit(1);
  }

  const passwordHash = await bcrypt.hash(newPassword, 10);

  const updated = await prisma.adminUser.update({
    where: { username },
    data: { passwordHash },
    select: { id: true, username: true, email: true, role: true },
  });

  console.log('密码修改成功:', updated);
}

main()
  .catch(console.error)
  .finally(() => prisma.$disconnect());
```

执行：

```bash
cd packages/api
pnpm exec tsx scripts/reset-password.ts Ethen 你的新强密码
```

> 脚本执行完毕后建议删除，避免新密码留在命令历史中。

## 安全建议

1. **上线后立即修改默认密码**：默认账号 `Ethen` / `Ethen112..` 已写入源码，必须第一时间修改
2. **使用强密码**：至少 12 位，包含大小写字母、数字与特殊符号，避免使用常见词
3. **限制管理后台访问**：
   - 通过 Vercel 部署时，可结合 Cloudflare Access 或 Vercel Password Protection 限制 `/login` 与管理路由的访问来源
   - 生产环境将管理后台绑定到内部域名，不对外公开
4. **最小权限原则**：日常运营创建 `editor` 角色账号，仅在需要时使用 `super_admin` 账号
5. **定期轮换密码与 JWT 密钥**：定期修改管理员密码，更换 `JWT_SECRET` 可使所有已签发 Token 失效
6. **监控异常登录**：关注 `/api/auth/admin-login` 接口的异常请求，项目已配置速率限制（默认 15 分钟 100 次）
7. **不要将生产环境 `.env` 提交到 Git**：`.env` 已在 `.gitignore` 中忽略，仅提交 `.env.example`

## 创建额外管理员示例

以下脚本演示如何创建新的管理员账号（幂等，按用户名查询已存在则跳过）：

```typescript
// packages/api/scripts/create-admin.ts
import bcrypt from 'bcryptjs';
import { PrismaClient } from '@prisma/client';

const prisma = new PrismaClient();

async function main() {
  const username = process.argv[2];
  const email = process.argv[3];
  const password = process.argv[4];
  const role = (process.argv[5] as 'super_admin' | 'editor') || 'editor';

  if (!username || !email || !password) {
    console.error('用法: tsx scripts/create-admin.ts <用户名> <邮箱> <密码> [角色]');
    console.error('角色: super_admin | editor（默认 editor）');
    process.exit(1);
  }

  // 检查是否已存在
  const existing = await prisma.adminUser.findUnique({ where: { username } });
  if (existing) {
    console.log(`管理员 ${username} 已存在，跳过创建`);
    return;
  }

  const passwordHash = await bcrypt.hash(password, 10);

  const admin = await prisma.adminUser.create({
    data: { username, email, passwordHash, role },
    select: { id: true, username: true, email: true, role: true },
  });

  console.log('管理员创建成功:', admin);
}

main()
  .catch(console.error)
  .finally(() => prisma.$disconnect());
```

执行示例：

```bash
cd packages/api

# 创建编辑账号
pnpm exec tsx scripts/create-admin.ts alice alice@mhvideo.com StrongPass123! editor

# 创建超级管理员账号
pnpm exec tsx scripts/create-admin.ts bob bob@mhvideo.com AnotherStrong456! super_admin
```

> 以上脚本为示例，需自行创建后执行。执行完毕后请妥善保管密码，建议使用密码管理工具。
