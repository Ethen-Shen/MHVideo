# 墨焕影视 - 对象存储（Cloudflare R2）

## 概述

墨焕影视使用 **Cloudflare R2** 作为对象存储，存放视频文件、封面图片与用户头像。R2 兼容 S3 API，项目通过 `@aws-sdk/client-s3` 与 `@aws-sdk/s3-request-presigner` 进行访问。

- **SDK**：AWS SDK v3（`@aws-sdk/client-s3`、`@aws-sdk/s3-request-presigner`）
- **实现文件**：`packages/api/src/lib/r2.ts`
- **上传控制器**：`packages/api/src/controllers/upload.controller.ts`
- **默认存储桶**：`mhvideo`

R2 客户端初始化（`r2.ts`）：

```typescript
const r2Client = new S3Client({
  region: 'auto',
  endpoint: process.env.R2_ENDPOINT!,
  credentials: {
    accessKeyId: process.env.R2_ACCESS_KEY_ID!,
    secretAccessKey: process.env.R2_SECRET_ACCESS_KEY!,
  },
});
```

## 前置条件

- **Cloudflare 账号**：已注册 [cloudflare.com](https://cloudflare.com)
- **已创建 R2 存储桶**：在 Cloudflare Dashboard → R2 中创建存储桶（默认名 `mhvideo`）
- **已配置公开访问域名**：为存储桶绑定自定义域名，用于公开读取文件

## 操作步骤

### 1. 创建 R2 存储桶

1. 登录 Cloudflare Dashboard → **R2 Object Storage**
2. 点击 **Create bucket**，输入存储桶名称（如 `mhvideo`），选择区域（建议 `auto`）
3. 点击 **Create** 完成创建

### 2. 创建 R2 API Token

1. 在 R2 页面点击 **Manage R2 API Tokens**（或 **API Tokens**）
2. 点击 **Create API Token**
3. 配置 Token：
   - **Token name**：如 `mhvideo-api`
   - **Permissions**：选择 **Object Read & Write**
   - **Specify bucket**：选择刚创建的存储桶（或所有桶）
4. 点击 **Create API Token**
5. **记录以下凭据**（仅显示一次）：

| 凭据 | 说明 | 对应环境变量 |
|------|------|-------------|
| Access Key ID | 访问密钥 ID | `R2_ACCESS_KEY_ID` |
| Secret Access Key | 访问密钥 | `R2_SECRET_ACCESS_KEY` |
| Endpoint | S3 端点 | `R2_ENDPOINT` |

Endpoint 格式为：`https://<账户ID>.r2.cloudflarestorage.com`

### 3. 配置存储桶公开访问域名

R2 存储桶默认不公开访问，需绑定自定义域名：

1. 进入存储桶详情页 → **Settings** → **Custom Domains**
2. 点击 **Connect Domain**，输入域名（如 `cdn.mhvideo.com`）
3. 按提示在 Cloudflare DNS 中添加 CNAME 记录（若域名已在 Cloudflare 管理，会自动添加）
4. 等待域名状态变为 **Active**

配置后，可通过 `https://<你的域名>/<对象Key>` 公开访问文件。

### 4. 填写环境变量

将以下变量配置到 `packages/api/.env`（本地）或 Vercel Environment Variables（生产）：

```bash
# R2 API 端点（从 API Token 创建页获取）
R2_ENDPOINT="https://<账户ID>.r2.cloudflarestorage.com"

# R2 访问密钥 ID
R2_ACCESS_KEY_ID="你的Access Key ID"

# R2 访问密钥
R2_SECRET_ACCESS_KEY="你的Secret Access Key"

# R2 存储桶名称
R2_BUCKET_NAME="mhvideo"

# R2 公开访问域名（自定义域名，不带末尾斜杠）
R2_PUBLIC_URL="https://cdn.mhvideo.com"
```

## 存储路径约定

`r2.ts` 中定义了统一的路径生成函数，文件按类型分目录存储：

| 类型 | 路径规则 | 生成函数 | 示例 |
|------|---------|---------|------|
| 视频 | `videos/{seriesId}/{episodeNumber}.{ext}` | `getVideoKey(seriesId, episodeNumber, ext)` | `videos/clxabc123/1.mp4` |
| 封面 | `covers/{seriesId}/{episodeNumber}.{ext}` | `getCoverKey(seriesId, episodeNumber, ext)` | `covers/clxabc123/1.jpg` |
| 头像 | `avatars/{userId}.{ext}` | `getAvatarKey(userId, ext)` | `avatars/clxuser456.jpg` |
| 其他 | `misc/{时间戳}-{原始文件名}` | 上传控制器内联生成 | `misc/1718700000000-poster.png` |

- 视频默认扩展名 `mp4`，封面默认 `jpg`，头像默认 `jpg`
- 实际扩展名取自上传文件原始名，若无则使用默认值
- 公开访问 URL 由 `R2_PUBLIC_URL` + `/` + key 拼接，例如：`https://cdn.mhvideo.com/videos/clxabc123/1.mp4`

## 上传方式

项目提供两种上传方式，均需通过 `auth`（登录）与 `admin`（管理员）中间件校验。

### 方式一：服务端直传

**接口**：`POST /api/admin/upload`

**认证**：需管理员 Token（Bearer）

**请求**：`multipart/form-data`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `file` | File | 是 | 上传的文件 |
| `type` | string | 是 | 类型：`video` / `cover` / `avatar` / 其他 |
| `seriesId` | string | video/cover 必填 | 系列 ID |
| `episodeNumber` | string | video/cover 必填 | 集数 |

**文件限制**：
- 最大 500MB
- 允许类型：`video/mp4`、`video/webm`、`video/quicktime`、`image/jpeg`、`image/png`、`image/webp`

**响应示例**：

```json
{
  "message": "上传成功",
  "data": {
    "key": "videos/clxabc123/1.mp4",
    "url": "https://cdn.mhvideo.com/videos/clxabc123/1.mp4"
  }
}
```

**调用示例**：

```bash
curl -X POST https://你的域名/api/admin/upload \
  -H "Authorization: Bearer <access_token>" \
  -F "file=@/path/to/video.mp4" \
  -F "type=video" \
  -F "seriesId=clxabc123" \
  -F "episodeNumber=1"
```

> **注意**：Vercel Serverless Function 单次执行有超时限制（项目配置为 10 秒），大文件直传易超时。大文件请使用方式二（预签名 URL）。

### 方式二：预签名 URL 上传

客户端先向后端获取预签名 URL，再直接将文件 PUT 到 R2，文件不经过应用服务器，适合大文件上传。

**接口**：`GET /api/admin/upload/presign`

**认证**：需管理员 Token（Bearer）

**Query 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `key` | string | 是 | 对象存储路径，如 `videos/clxabc123/1.mp4` |
| `contentType` | string | 是 | 文件 MIME 类型，如 `video/mp4` |
| `expiresIn` | number | 否 | URL 有效期（秒），默认 3600 |

**响应示例**：

```json
{
  "message": "获取预签名URL成功",
  "data": {
    "url": "https://<账户ID>.r2.cloudflarestorage.com/mhvideo/videos/clxabc123/1.mp4?X-Amz-..."
  }
}
```

**调用示例**：

```bash
# 1. 获取预签名 URL
curl -G https://你的域名/api/admin/upload/presign \
  -H "Authorization: Bearer <access_token>" \
  --data-urlencode "key=videos/clxabc123/1.mp4" \
  --data-urlencode "contentType=video/mp4"

# 2. 使用预签名 URL 直接上传文件到 R2
curl -X PUT "<返回的预签名URL>" \
  -H "Content-Type: video/mp4" \
  --data-binary @/path/to/video.mp4
```

上传成功后，文件即可通过 `R2_PUBLIC_URL` + key 公开访问。

### 用户头像上传

普通用户（非管理员）头像上传使用独立接口：

**接口**：`POST /api/user/avatar`

**认证**：需用户登录 Token（Bearer）

**请求**：`multipart/form-data`，字段 `avatar`

存储路径由 `getAvatarKey(userId, ext)` 生成，如 `avatars/clxuser456.jpg`。

## 常见问题

### Q1：上传报错 "Access Denied" 或 403？

- 检查 `R2_ACCESS_KEY_ID` 与 `R2_SECRET_ACCESS_KEY` 是否正确
- 检查 API Token 权限是否为 **Object Read & Write**
- 检查 Token 是否绑定到正确的存储桶

### Q2：文件上传成功但无法通过 URL 访问？

- 确认存储桶已绑定自定义域名且状态为 Active
- 确认 `R2_PUBLIC_URL` 配置正确（不带末尾斜杠）
- R2 默认存储桶对象不公开，必须通过自定义域名访问

### Q3：`R2_ENDPOINT` 如何获取？

在 Cloudflare Dashboard 创建 R2 API Token 时，页面会显示 Endpoint，格式为 `https://<账户ID>.r2.cloudflarestorage.com`。账户 ID 可在 Cloudflare Dashboard 右侧栏找到。

### Q4：大文件上传超时？

Vercel Serverless Function 有执行时长限制。大文件请使用预签名 URL 方式（`GET /api/admin/upload/presign`），由客户端直接 PUT 到 R2，绕过应用服务器。

### Q5：预签名 URL 过期了怎么办？

预签名 URL 默认有效期 3600 秒（1 小时），可通过 `expiresIn` 参数自定义。过期后需重新调用 `/api/admin/upload/presign` 获取新 URL。

### Q6：如何删除已上传的文件？

当前 `r2.ts` 仅实现上传与预签名功能，未提供删除接口。如需删除文件，可在 Cloudflare R2 Dashboard 手动删除，或扩展 `r2.ts` 使用 `DeleteObjectCommand` 实现。
