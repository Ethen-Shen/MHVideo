# 墨焕影视 - 数据库（Prisma Postgres）

## 概述

墨焕影视使用 **Prisma Postgres**（部署在 Vercel 上的 PostgreSQL）作为业务数据库，通过 **Prisma ORM** 进行数据建模与访问。

- **数据库引擎**：PostgreSQL（Vercel Postgres）
- **ORM**：Prisma 5.14+
- **Schema 文件**：`packages/api/prisma/schema.prisma`
- **Prisma Client**：`@prisma/client`，由 `prisma generate` 生成

Schema 中 `datasource db` 配置如下：

```prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}
```

数据库连接字符串通过环境变量 `DATABASE_URL` 注入。

## 前置条件

- **Vercel 账号**：拥有项目所在 Vercel 团队的访问权限
- **Node.js >= 18 / pnpm >= 8**：本地执行迁移命令所需
- **项目依赖已安装**：在项目根目录执行过 `pnpm install`

## 操作步骤

### 1. 在 Vercel Dashboard 创建 Postgres 数据库

1. 进入 Vercel 项目页面，切换到 **Storage** 标签
2. 点击 **Create Database** → 选择 **Postgres**（即 Prisma Postgres）
3. 输入数据库名称（如 `mhvideo`），选择部署区域（建议与 Vercel 项目同区域以降低延迟）
4. 点击 **Create** 完成创建

### 2. 获取 DATABASE_URL

数据库创建完成后，在数据库详情页可获取连接字符串：

- 进入数据库详情页 → **.env.local** 或 **Connect** 标签
- 复制 `DATABASE_URL` 的值，格式为：

```
postgresql://用户名:密码@主机:5432/数据库名?schema=public
```

> Vercel 同时提供连接池地址（`?pgbouncer=true&connection_limit=1`）与直连地址。Serverless 环境建议使用连接池地址；执行 Prisma 迁移时建议使用直连地址（不带 pgbouncer 参数）。

### 3. 配置到环境变量

**生产环境（Vercel）**：

在 Vercel Dashboard → **Settings** → **Environment Variables** 中添加：

| 变量名 | 值 |
|--------|----|
| `DATABASE_URL` | 上一步复制的连接字符串 |

**本地开发**：

```bash
cd packages/api
cp .env.example .env
# 编辑 .env，将 DATABASE_URL 替换为 Vercel Postgres 连接字符串
```

### 4. 生成 Prisma Client

```bash
pnpm --filter @mhvideo/api exec prisma generate
```

该命令根据 `schema.prisma` 生成类型安全的 Prisma Client。Vercel 构建时 `vercel-build` 脚本会自动执行此命令。

### 5. 执行数据库迁移

**生产环境**（应用已有迁移）：

```bash
pnpm --filter @mhvideo/api exec prisma migrate deploy
```

`migrate deploy` 会读取 `prisma/migrations` 目录下所有迁移记录，按顺序应用尚未执行的迁移，不会交互式提问，适合 CI/CD 与生产部署。

**本地开发**（修改 schema 后创建新迁移）：

```bash
pnpm --filter @mhvideo/api exec prisma migrate dev --name <迁移名称>
```

`migrate dev` 会对比 schema 与数据库差异，生成新的迁移文件，并重新生成 Prisma Client。

### 6. 执行 Seed 创建管理员

```bash
pnpm --filter @mhvideo/api exec prisma db seed
```

该命令执行 `packages/api/prisma/seed.ts`，幂等创建默认超级管理员账号（用户名 `Ethen`、角色 `super_admin`）。重复执行不会报错也不会重复创建。详见 [管理员账号.md](./管理员账号.md)。

## 数据模型说明

Schema 定义在 `packages/api/prisma/schema.prisma`，核心模型如下：

### User（用户）

普通终端用户，支持邮箱、手机号、微信/抖音/TikTok OpenID 多种登录方式。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String | 主键，cuid |
| `email` | String? | 邮箱，唯一 |
| `phone` | String? | 手机号，唯一 |
| `passwordHash` | String? | 密码哈希（bcrypt） |
| `nickname` | String | 昵称 |
| `avatarUrl` | String? | 头像地址 |
| `locale` | String | 语言，默认 `zh-CN` |
| `themePreference` | String | 主题偏好，默认 `light` |
| `status` | String | 状态：`active` / `banned` |
| `wechatOpenid` / `douyinOpenid` / `tiktokOpenid` | String? | 各平台 OpenID，唯一 |
| `createdAt` / `updatedAt` | DateTime | 创建/更新时间 |

关联：`Comment`、`Like`、`Favorite`、`FavoriteFolder`、`Rating`、`WatchHistory`。表名 `users`。

### AdminUser（管理员）

管理后台账号。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String | 主键，cuid |
| `email` | String | 邮箱，唯一 |
| `passwordHash` | String | 密码哈希（bcrypt） |
| `role` | String | 角色：`super_admin` / `editor`，默认 `editor` |
| `createdAt` | DateTime | 创建时间 |

表名 `admin_users`。

### Category（分类）

视频与系列的分类。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String | 主键 |
| `name` | String | 分类名 |
| `iconUrl` | String? | 图标地址 |
| `sortOrder` | Int | 排序，默认 0 |
| `createdAt` | DateTime | 创建时间 |

关联：`Video[]`、`Series[]`。表名 `categories`。

### Series（系列）

剧集/合集，包含多集视频。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String | 主键 |
| `title` | String | 标题 |
| `description` | String? | 描述 |
| `coverUrl` | String? | 封面地址 |
| `status` | String | 状态：`published` / `draft`，默认 `draft` |
| `categoryId` | String? | 关联分类 |
| `createdAt` / `updatedAt` | DateTime | 创建/更新时间 |

关联：`Category?`、`Video[]`、`SeriesTag[]`。表名 `series`。

### Video（视频）

单集视频。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String | 主键 |
| `title` | String | 标题 |
| `description` | String? | 描述 |
| `coverUrl` | String? | 封面地址 |
| `videoUrl` | String? | 视频地址 |
| `duration` | Int | 时长（秒），默认 0 |
| `episodeNumber` | Int | 集数，默认 1 |
| `viewCount` | Int | 播放量，默认 0 |
| `likeCount` | Int | 点赞数，默认 0 |
| `status` | String | 状态：`published` / `draft`，默认 `draft` |
| `seriesId` | String? | 关联系列 |
| `categoryId` | String? | 关联分类 |
| `createdAt` / `updatedAt` | DateTime | 创建/更新时间 |

索引：`[seriesId, episodeNumber]`、`[status]`、`[categoryId]`。关联：`Series?`、`Category?`、`Comment[]`、`Like[]`、`Favorite[]`、`Rating[]`、`WatchHistory[]`、`VideoTag[]`。表名 `videos`。

### Comment（评论）

视频评论，支持嵌套回复。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String | 主键 |
| `content` | String | 评论内容 |
| `parentId` | String? | 父评论 ID（回复） |
| `userId` | String | 评论用户 |
| `videoId` | String | 关联视频 |
| `createdAt` / `updatedAt` | DateTime | 创建/更新时间 |

自关联 `CommentReplies` 实现评论回复树。索引：`[videoId]`、`[userId]`。表名 `comments`。

### 其他模型

| 模型 | 表名 | 说明 |
|------|------|------|
| `SeriesTag` | `series_tags` | 系列标签，`[seriesId, tag]` 唯一 |
| `VideoTag` | `video_tags` | 视频标签，`[videoId, tag]` 唯一 |
| `Like` | `likes` | 点赞，`[userId, videoId]` 唯一 |
| `FavoriteFolder` | `favorite_folders` | 收藏夹 |
| `Favorite` | `favorites` | 收藏，`[userId, videoId]` 唯一 |
| `Rating` | `ratings` | 评分（1-5），`[userId, videoId]` 唯一 |
| `WatchHistory` | `watch_history` | 观看历史，记录进度与完成状态，`[userId, videoId]` 唯一 |

## 本地开发

### 1. 配置环境变量

```bash
cd packages/api
cp .env.example .env
```

编辑 `.env`，填入本地或 Vercel Postgres 的 `DATABASE_URL`。

### 2. 创建迁移并同步数据库

```bash
# 在 packages/api 目录下
pnpm exec prisma migrate dev --name init
```

该命令会：
- 创建 `prisma/migrations` 迁移目录
- 应用迁移到数据库
- 重新生成 Prisma Client

### 3. 使用 Prisma Studio 查看数据

```bash
pnpm --filter @mhvideo/api exec prisma studio
```

Prisma Studio 会在 `http://localhost:5555` 启动可视化数据管理界面，可直接浏览与编辑各表数据。

### 4. 重置数据库（谨慎）

```bash
pnpm --filter @mhvideo/api exec prisma migrate reset
```

该命令会删除所有数据并重新应用全部迁移，随后自动执行 seed。**仅限开发环境使用**。

## 常见问题

### Q1：`prisma migrate deploy` 报错 "migration failed"？

- 确认 `DATABASE_URL` 指向的数据库可访问
- 若使用连接池地址（含 `pgbouncer=true`），迁移时请改用直连地址
- 检查 `prisma/migrations` 目录是否存在且迁移文件完整

### Q2：Serverless 环境下数据库连接数耗尽？

Vercel Serverless Function 每次调用可能创建新连接。建议：
- 使用 Vercel Postgres 提供的连接池地址（含 `?pgbouncer=true&connection_limit=1`）作为 `DATABASE_URL`
- Prisma Client 在 Serverless 环境下会复用全局实例（项目 `packages/api/src/lib/prisma.ts` 已实例化为全局 `prisma`）

### Q3：`prisma db seed` 报错？

- 确认 `packages/api/package.json` 中已配置 `prisma.seed` 字段
- 确认 `packages/api/prisma/seed.ts` 文件存在
- 确认 `DATABASE_URL` 已正确配置且数据库迁移已执行

### Q4：修改 schema 后如何同步？

```bash
cd packages/api
pnpm exec prisma migrate dev --name <描述性名称>
```

这会生成新迁移文件、应用到数据库、重新生成 Client。提交迁移文件到 Git 后，生产环境通过 `prisma migrate deploy` 应用。

### Q5：Prisma Client 类型未更新？

执行 `pnpm --filter @mhvideo/api exec prisma generate` 重新生成。若 TypeScript 仍报错，重启 IDE 或 TS Server。
