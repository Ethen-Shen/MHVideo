# 墨焕影视 - Vercel 部署指南

## 概述

墨焕影视采用 Vercel 进行部署，技术栈为：

- **monorepo**：pnpm workspace，包含 `admin`（Vue 3 管理后台）、`api`（Express 后端）、`miniapp`（小程序）、`shared`（共享代码）四个包
- **后端**：Express，通过 Vercel Serverless Functions 运行（入口 `api/index.ts`）
- **前端**：Vue 3 + Vite 构建的管理后台，静态托管
- **数据库**：Vercel Postgres（Prisma ORM）
- **对象存储**：Cloudflare R2

`vercel.json` 已配置好构建命令、路由规则与函数内存。Vercel 会自动识别 `api/index.ts` 作为 Serverless Function 入口，将 `/api/*` 请求路由到后端，其余请求回退到管理后台静态资源。

## 前置条件

- **Node.js**：>= 18.0.0
- **pnpm**：>= 8.0.0（项目使用 pnpm@11.5.1）
- **Vercel 账号**：已注册 [vercel.com](https://vercel.com) 账号
- **GitHub 仓库**：项目代码已推送到 GitHub 仓库
- **Cloudflare 账号**：已创建 R2 存储桶并获取 API Token（详见 [对象存储.md](./对象存储.md)）

## 操作步骤

### 1. 导入仓库到 Vercel

1. 登录 [Vercel Dashboard](https://vercel.com/dashboard)
2. 点击 **Add New...** → **Project**
3. 在 **Import Git Repository** 中找到墨焕影视的 GitHub 仓库，点击 **Import**
4. Framework Preset 选择 **Other**（项目为 monorepo，由自定义构建脚本处理）

### 2. 配置构建命令与输出目录

Vercel 项目设置中确认以下配置（与 `vercel.json` 一致，通常会自动读取）：

- **Build Command**：`npm run vercel-build`
- **Output Directory**：`.`
- **Install Command**：留空（构建脚本中已包含 `pnpm install`）

`vercel-build` 脚本（定义在根 `package.json`）会依次执行：

```bash
pnpm install
pnpm --filter @mhvideo/api exec prisma generate
pnpm --filter @mhvideo/api build
pnpm --filter @mhvideo/admin build
```

即：安装依赖 → 生成 Prisma Client → 编译后端 TypeScript → 构建管理后台前端。

### 3. 配置环境变量

在 Vercel Dashboard → **Settings** → **Environment Variables** 中，添加以下变量（参考 `packages/api/.env.example`）：

#### 数据库

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `DATABASE_URL` | Vercel Postgres 连接字符串 | `postgresql://user:password@host:5432/mhvideo?schema=public` |

#### JWT 认证

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `JWT_SECRET` | JWT 密钥（至少 32 位随机字符串） | `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"` 生成 |
| `JWT_EXPIRES_IN` | Access Token 有效期 | `7d` |
| `JWT_REFRESH_EXPIRES_IN` | Refresh Token 有效期 | `30d` |

#### Cloudflare R2 存储

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `R2_ENDPOINT` | R2 API 端点 | `https://<账户ID>.r2.cloudflarestorage.com` |
| `R2_ACCESS_KEY_ID` | R2 访问密钥 ID | - |
| `R2_SECRET_ACCESS_KEY` | R2 访问密钥 | - |
| `R2_BUCKET_NAME` | R2 存储桶名称 | `mhvideo` |
| `R2_PUBLIC_URL` | R2 公开访问域名 | `https://cdn.example.com` |

#### 文件上传

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `UPLOAD_MAX_FILE_SIZE` | 最大文件大小（字节） | `524288000`（500MB） |
| `UPLOAD_ALLOWED_TYPES` | 允许的文件类型 | `video/mp4,video/webm,video/quicktime,image/jpeg,image/png,image/webp` |

#### 速率限制

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `RATE_LIMIT_WINDOW_MS` | 速率限制时间窗口（毫秒） | `900000`（15 分钟） |
| `RATE_LIMIT_MAX` | 时间窗口内最大请求数 | `100` |

#### 运行环境

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `NODE_ENV` | 运行环境 | `production` |

> **提示**：建议将所有环境变量的 Environment 选择为 **Production**、**Preview**、**Development** 全部勾选，或至少勾选 Production。

### 4. 连接 Vercel Postgres 数据库

1. 在 Vercel 项目页面，切换到 **Storage** 标签
2. 点击 **Create Database** → 选择 **Postgres**（Prisma Postgres）
3. 输入数据库名称（如 `mhvideo`），选择区域（建议靠近用户），点击 **Create**
4. 创建完成后，进入数据库详情页，点击 **Connect to Project** 将数据库连接到当前项目
5. Vercel 会自动注入 `DATABASE_URL` 等环境变量到项目中

> 也可在数据库详情页的 **.env.local** 标签下手动复制 `DATABASE_URL`，粘贴到项目的 Environment Variables 中。

详细的 Prisma Postgres 与 Vercel 集成说明参见 [数据库.md](./数据库.md)。

### 5. 执行数据库迁移

数据库连接后，需要执行 Prisma 迁移以创建表结构。

**方式一：本地执行（推荐）**

1. 在本地项目根目录复制环境变量文件并填入生产数据库地址：

```bash
cd packages/api
cp .env.example .env
# 编辑 .env，将 DATABASE_URL 替换为 Vercel Postgres 的连接字符串
```

2. 执行迁移：

```bash
pnpm --filter @mhvideo/api exec prisma migrate deploy
```

`prisma migrate deploy` 会应用 `prisma/migrations` 目录下所有未执行的迁移，适用于生产环境。

**方式二：Vercel 构建时执行**

如希望在 Vercel 构建阶段自动执行迁移，可在 Vercel 项目的 Build Command 后追加迁移命令（需确保 `DATABASE_URL` 已配置）。但更推荐在本地手动执行，以便观察迁移输出。

### 6. 创建默认管理员账号

迁移完成后，执行 Prisma seed 脚本创建默认超级管理员：

```bash
pnpm --filter @mhvideo/api exec prisma db seed
```

该命令会创建默认超级管理员账号（用户名 `Ethen`、密码 `Ethen112..`、角色 `super_admin`）。详见 [管理员账号.md](./管理员账号.md)。

> **安全提醒**：上线后请立即修改默认密码！

### 7. 部署

1. 环境变量与数据库配置完成后，在 Vercel Dashboard 点击 **Redeploy** 或推送新代码到 GitHub 触发部署
2. 部署完成后，Vercel 会分配一个 `*.vercel.app` 域名，访问该域名即可打开管理后台
3. 访问 `https://<你的项目>.vercel.app/login` 进入登录页，使用默认管理员账号登录

## 自定义域名

1. 在 Vercel 项目页面 → **Settings** → **Domains**
2. 输入你的域名（如 `admin.mhvideo.com`），点击 **Add**
3. 按提示到域名注册商处添加 DNS 记录：
   - 类型：`CNAME`
   - 名称：`admin`（或对应子域名）
   - 值：`cname.vercel-dns.com`
4. DNS 生效后（通常几分钟到几小时），Vercel 会自动签发 SSL 证书
5. 建议在域名设置中将该域名设为 **Primary Domain**，并将 `*.vercel.app` 域名重定向到自定义域名

## 常见问题

### Q1：构建失败，提示 `prisma generate` 找不到 schema？

确保 Vercel 的工作目录识别正确。项目根目录的 `vercel-build` 脚本已使用 `pnpm --filter @mhvideo/api exec prisma generate` 指定工作空间，无需额外配置。如仍失败，检查 `pnpm-workspace.yaml` 是否包含 `packages/*`。

### Q2：部署后访问 API 返回 404？

检查 `vercel.json` 中的路由配置。`/api/(.*)` 应路由到 `/api/index.ts`。确认 `api/index.ts` 文件存在于项目根目录（非 packages 目录），它作为 Serverless Function 入口导出 Express app。

### Q3：数据库连接失败？

- 确认 `DATABASE_URL` 环境变量已正确配置且环境勾选了 Production
- 确认 Vercel Postgres 数据库已 **Connect to Project**
- 检查连接字符串格式是否为 `postgresql://user:password@host:5432/dbname?schema=public`

### Q4：管理后台页面空白？

- 检查构建日志，确认 `pnpm --filter @mhvideo/admin build` 执行成功
- 确认 `packages/admin/dist` 目录已生成
- `vercel.json` 中 `outputDirectory` 为 `.`，路由规则将非 `/api/*` 请求回退到 `packages/admin/dist/$1`

### Q5：文件上传失败？

- 确认 R2 相关环境变量（`R2_ENDPOINT`、`R2_ACCESS_KEY_ID`、`R2_SECRET_ACCESS_KEY`、`R2_BUCKET_NAME`、`R2_PUBLIC_URL`）已正确配置
- 确认 R2 存储桶已创建且 API Token 有读写权限
- 详见 [对象存储.md](./对象存储.md)

### Q6：Vercel Serverless Function 超时？

`vercel.json` 中配置了函数 `maxDuration: 10`（秒）。如大文件上传等操作超时，可在 Vercel Pro/Enterprise 计划中提升超时上限（最高 60 秒 / 300 秒）。注意 Serverless 环境下不建议通过服务端直传大文件，应使用预签名 URL 方式（见 [对象存储.md](./对象存储.md)）。
